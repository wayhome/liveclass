Using 2 Editors Because Xcode Is Dumb

作者: ingve | 评分: 48 | 评论数: 17 | 发布时间: 2025-01-23 12:24
文章摘要

本文讨论了在Xcode 16中使用本地Swift包进行自动补全开发时遇到的问题。作者之前使用Xcode 15，通过拖放方式将本地包添加到项目中，但Xcode 16要求通过指定路径（如`../../PACKAGE_NAME/`）添加本地依赖。新方法存在多个问题：文件操作（如添加、删除、重命名文件）后项目无法自动更新，导致构建错误；无法运行包的测试；也不能同时在两个Xcode窗口中打开包和项目。作者感到沮丧，因为这些变化严重影响了开发效率。尽管可以通过命令行工具和SourceKit LSP在其他编辑器（如Emacs或Neovim）中编辑和测试包，但仍对Xcode的强制更新和问题感到不满，建议开发者准备备用编辑器以应对类似问题。

评论摘要
主要讨论点：Xcode 及其他代码编辑器的使用体验和优缺点 不同观点： • PaulHoule：认为使用备用编辑器（如 vi）很正常，vi 的优势在于几乎所有 Linux 系统上都存在，具有很高的可靠性。他对 vi 的使用仅限于基本功能，并且提到 Emacs 的操作（如 ^S 搜索）在命令行中存在冲突问题。 • deergomoo：对 Xcode 的体验非常不满，指出其在构建项目、崩溃问题、缺失文本编辑功能（如多光标）、查找替换功能差以及 Swift 类型检查速度慢等方面的问题。此外，提到即使在简单项目中，Xcode 仍会出现重命名文件失败、断点停止工作等问题。 • jessekv：引用了 App Store 上对 Xcode 的评论，调侃其虽然不好用，但仍比下一代更佳。 • joshstrange：同样对 Xcode 不满，认为其速度慢、自动补全和错误报告延迟严重，更倾向于使用 JetBrains 的 AppCode。即使在 AppCode 中，某些任务仍需返回 Xcode 完成。 • wilkystyle：从更广泛的角度讨论了苹果系统的更新问题，指出每次更新后都会出现一些以前正常功能被破坏的情况，并列举了多个例子，如 macOS 的系统通知问题和 iOS 的 Siri 设置问题。 • iamdamian：表示 Xcode 的问题使他难以进入 macOS/iOS 编程领域，尽管他对构建原生应用感兴趣。 • simonbarker87：因为 Xcode 的笨重、缓慢、不透明以及 Swift 的错误消息不清晰和苹果开发文档差劲，放弃了 iOS 开发。 • jtrueb：由于 Xcode 的速度和响应时间问题，主要使用 Cursor 编辑器，仅在必须时切换回 Xcode，并认为 LLM 编辑器的响应速度优于 Xcode。 • chuckadams：尽管喜欢 Mac 硬件和部分软件，但认为 Xcode 只是一个依赖工具，使用体验很差。 • phreack：指出 Xcode 多年来变得越来越差，工具栏图标不明确，操作流程繁琐，使得日常工作不便。 • nntwozz：用“用链锯理发”来比喻 Xcode 的笨重和不适用。 • lawgimenez：同时使用 Xcode 和 Android Studio，更喜欢后者的编码体验，指出 Xcode 容易因为简单的 git 切换操作导致项目构建失败。 • mhd：提到 Xcode 的早期版本问题较多，很多编辑工作需借助更稳定和功能更强的编辑器完成。 • st3fan：讨论了项目配置文件（Package.json）的正确设置问题，指出正确的项目结构可以避免一些 Xcode 问题。 • 0xferruccio：认为 Xcode 很差，希望苹果能开放 VS Code 和 Vim 等编辑器的扩展支持，以改善开发者体验。 补充讨论： - Xcode 的速度和性能是多数开发者抱怨的焦点，尤其是自动补全和错误报告的延迟。 - Xcode 的用户界面和工具栏设计被认为不直观，图标不明确，工具发现性差。 - 使用备用编辑器（如 vi 和 Emacs）是部分开发者的常见做法，尤其是当他们需要在不同系统间切换时。 - 苹果系统的更新问题也被提及，更新后某些功能被破坏使得用户体验下降。 - 对 Swift 语言本身的喜爱与 Xcode 的糟糕体验形成对比，部分开发者因 Xcode 问题放弃了 iOS 开发。 争议焦点： - Xcode 的性能、用户界面设计和可靠性是主要争议点，部分开发者认为其难以使用，而另一部分开发者则习惯了使用备用编辑器来避免这些问题。

Minecraft with object impermanence

作者: sohkamyung | 评分: 129 | 评论数: 21 | 发布时间: 2025-01-19 22:03
文章摘要

Most Influential Papers in Computer Science History

文章摘要
