UK's hardware talent is being wasted

作者: sebg | 评分: 393 | 评论数: 62 | 发布时间: 2025-01-19 23:52
文章摘要

文章指出，英国顶尖大学（如帝国理工、牛津、剑桥）培养的硬件工程人才在毕业后面临严重的经济困境，工资远低于国际水平，尤其是与硅谷相比。许多毕业生因薪资低而转行至金融或咨询业，导致人才浪费。英国硬件工程领域投资不足，传统企业创新乏力，造成人才外流和创新停滞。文章强调，这不仅是工资差距，更是国家层面的人才错配，可能影响科技创新和国家安全。然而，硬件工程领域也存在巨大机遇，尤其是对于有雄心的初创企业。英国应抓住时机，充分利用本地人才，避免错失像ARM或Dyson这样的成功案例。现在行动，尚有机会改变局面。

评论摘要
主要讨论点：硬件创业与软件创业的差异、地域对创业的影响、英国与其他国家（尤其是美国）在科技创业环境上的对比 不同观点： • **硬件创业比软件创业更难**：[tomhoward] 认为硬件创业的难度远高于软件创业，主要是由于硬件开发的成本高、周期长、复杂性大，且难以获得投资者的关注和支持。相比之下，软件创业可以在较短时间内看到增长和收入，更容易吸引投资。 • **工程与制造不能分离**：[Animats] 强调工程与制造的紧密联系，认为如果工程与制造分离，会导致设计成本过高。通过实例说明即使美国公司也倾向于将工程团队设在制造工厂附近，以提高效率。 • **英国工程职业路径有限**：[refrigerator] 指出英国的硬件工程职业路径有限，薪酬低，导致许多有才华的工程毕业生转向软件工程、金融或咨询行业。 • **英国整体职业市场问题**：[incog_nit0] 认为问题不仅存在于硬件行业，英国整体职业市场的薪酬和职业发展机会都不如美国，尤其是在非金融行业。 • **英国缺乏风险资本和创业文化**：[r_thambapillai] 指出英国的风险资本不足，投资者更关注价格而非公司潜力。同时，英国创业文化中的野心和冒险精神相对较小，导致人才流失。 • **宏观经济因素影响职业选择**：[awanderingmind] 认为收入和生活质量是职业选择的重要因素，许多人选择去美国或进入金融行业是为了追求更高的收入和更好的生活质量。 • **英国历史上的科技投资不足**：[alexisread] 回顾了英国在多个科技领域的历史投资不足，指出英国在科技投资上的长期问题。 • **硬件创业风险和物流挑战**：[remus] 强调硬件创业的固有风险和物流挑战，认为硬件创业的每个步骤都比软件创业更复杂、成本更高，风险更大。 • **英国创业环境和资本问题**：[alephnerd] 认为英国的创业环境存在根本问题，风险资本不足，许多英国公司的工程和产品团队位于其他国家，英国本土的投资机会有限。 补充讨论： • **英国经济和政治问题**：[tidenly] 和 [James_K] 提到英国的政治和经济问题对创业环境和整体经济的影响，认为英国的经济和政治环境不稳定，导致人才流失和经济困境。 • **职业选择的地域差异**：[t43562] 提到软件行业的职业机会和薪酬在英国不同地区存在差异，且英国社会对科技行业的尊重程度不如金融行业，这影响了职业选择和创业环境。 争议焦点： • **硬件创业是否比软件创业更难**：一些评论认为硬件创业的固有风险和物流挑战使其比软件创业更难，而另一些评论则认为这取决于具体的创业目标和市场策略。 • **英国创业环境的问题根源**：一些评论认为英国的创业环境问题主要是由于风险资本不足和投资文化保守，而另一些评论则认为这与英国整体的经济和政治环境有关。 总结： 讨论中主要围绕硬件创业与软件创业的难度对比、英国与其他国家在科技创业环境上的差异、以及英国经济和政治环境对创业的影响展开。不同的评论者从个人经历、市场观察和历史回顾等角度提供了丰富的论据和例子，揭示了硬件创业面临的固有挑战，以及英国在科技创业环境上的结构性问题。

文章摘要

Parinfer是一种简化Lisp编程的编辑器模式，旨在通过自动调整括号和缩进来降低Lisp编辑的复杂性，尤其对新手更友好。它通过定义括号与缩进的关系，使得代码格式自动符合约定，并支持在缩进变化时影响表达式嵌套。Parinfer还允许在无快捷键情况下实现Paredit的部分功能，同时保持现有编辑器的强大功能。Lisp代码中，缩进用于快速浏览内容，而括号则用于详细检查嵌套结构，Parinfer通过自动管理这些细节，提升了编码体验。该工具旨在解决传统Lisp编辑工具学习曲线陡峭的问题，同时不牺牲Lisp的强大特性。

Rhai: An embedded scripting language for Rust

作者: airstrike | 评分: 84 | 评论数: 12 | 发布时间: 2025-01-17 15:40
文章摘要

Rhai 是一个为 Rust 设计的内嵌脚本语言和评估引擎，旨在为应用添加安全、易用的脚本功能。它支持所有 Rust 支持的 CPU 和操作系统目标，包括 WebAssembly 和 no-std 环境，最低需要 Rust 1.66.0 版本。Rhai 具有动态类型、高效评估、与 Rust 函数和类型的紧密集成等特点，支持常见数据类型和序列化/反序列化。它提供了一个插件系统，支持函数重载、操作符重载、闭包和面向对象编程，还可以通过 AST 进行优化和多次评估。Rhai 引擎是沙盒化的，能防止恶意攻击，并支持手动终止脚本运行。此外，它还提供了调试接口和 Miri 支持，具有高度可定制性，允许用户禁用某些语言特性或自定义语法。文档和在线 playground 可在项目网站和 Rhai Book 中查阅。Rhai 采用 Apache-2.0 和 MIT 双重许可。

TypeScript enums: use cases and alternatives

文章摘要

这篇文章深入探讨了TypeScript中的枚举（enums），包括其工作原理、使用场景及替代方案，并给出不同情况下的推荐用法。 1. **枚举的工作原理**： - TypeScript枚举定义了一个对象，将成员键映射到成员值。 - 枚举也是一个仅包含成员值的类型。 2. **使用场景**： - 枚举可以是对象，类似对象字面量。 - 枚举可以作为类型使用，限制变量只能取枚举成员值。 - 支持穷尽性检查，确保处理了所有可能的枚举值。 3. **替代方案**： - 使用对象字面量和联合类型可以实现类似枚举的功能。 - 通过`UnsupportedValueError`类在运行时进行检查。 4. **枚举成员的枚举**： - 编译成JavaScript后，枚举是一个立即调用的函数，添加属性到对象。 文章建议根据具体需求选择使用枚举或其替代方案，并在代码中结合`UnsupportedValueError`进行运行时检查。

评论摘要
主要讨论点：TypeScript中枚举（Enums）的使用及其替代方案的优缺点 不同观点： • **不推荐使用Enums**： - [skrebbel], [msoad], [ivanjermakov], [bluelightning2k]等用户认为Enums在当前TypeScript中已经过时，因为引入了如字符串字面量和联合类型（`type Status = "Active" | "Inactive"`）以及`as const`等新特性，这些新特性可以更好地替代Enums。 - Enums会在运行时生成代码，增加了不必要的运行时开销，而联合类型和常量对象不会。 - Enums无法在类型声明文件（`.d.ts`）中使用，可能会影响打包后的代码体积，甚至导致服务器端代码泄露到客户端。 • **支持使用Enums**： - [conaclos]认为Enums提供了更强的类型安全性，因为不同枚举类型的值不能互相赋值。此外，一些打包工具可以对Enums进行内联优化，使其在某些情况下优于联合类型。 - [Aeolun]表示自己像使用Java枚举一样使用TS Enums，从未遇到问题，认为Enums依然有其使用价值。 • **对Enums的历史和演进的反思**： - [nick_wolf]认为TypeScript在早期加入了Enums可能是个错误，应该让社区自然演化出最佳实践，而不是一开始就设计好所有特性。 - [rednafi]认为TypeScript增加了许多不必要的复杂性，尽管在某些情况下这些复杂性是可以忍受的，但总体上增加了理解和使用难度。 补充讨论： • **联合类型和常量对象的优点**： - [chpatrick], [estsauver], [MortyWaves]等用户支持使用联合类型和常量对象，认为它们更简洁、类型安全且在调试时更直观。 - 联合类型和常量对象不会生成额外的运行时代码，且在某些场景下（如条件渲染）更灵活。 • **争议焦点**： - Enums是否应该被标记为弃用，尤其是在有更好的替代方案的情况下。 - Enums的运行时开销和类型安全性是否足以抵消其使用上的便利性。 • **其他值得注意的讨论点**： - 用户[forty]提供了一个自定义的Enum实现，提供了类型守卫和迭代功能，以方便从原生TypeScript Enum过渡。 - 用户[bluelightning2k]提到Enums不能用于接口声明文件或作为类型导入，这对打包和代码部署有影响。 总结：讨论主要围绕是否继续使用TypeScript Enums及其替代方案的优缺点展开，涉及到类型安全性、运行时开销、代码简洁性和社区演进等方面。
