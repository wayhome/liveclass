Coping with dumb LLMs using classic ML

本文介绍了一种利用本地LLM（大型语言模型）判断产品搜索相关性的方法，目标是替代昂贵的人工标注或OpenAI服务。通过对比LLM与人类标注者在产品相关性判断上的表现，验证LLM的可靠性。作者设计了一系列实验，让LLM基于产品名称、分类、描述等属性做出判断，并引入"Neither"选项允许模型不做出决定。实验结果显示，允许模型不双重检查并选择"Neither"时，预测准确率较高（如产品名称的准确率达到90.76%），但覆盖率较低（仅11.9%的判断对）。通过这些实验，作者尝试优化LLM在搜索相关性判断中的表现，以实现更高效的搜索质量调整。

文章摘要

文章介绍了19世纪末德国数学家卡尔·魏尔斯特拉斯（Karl Weierstrass）提出的一个颠覆传统微积分的“锯齿状怪物”函数。这一函数挑战了微积分的基本概念，尤其是连续性和可微性。传统上，数学家认为连续函数在绝大多数点上是可微的，只有有限个点可能不可微。然而，魏尔斯特拉斯的函数无论怎样放大都呈现出锯齿状，且在所有点上都不可微，这打破了当时的常识。这一发现引发了法国数学家的愤怒和恐惧，但最终帮助微积分建立了更坚实的基础，推动了数学的发展。魏尔斯特拉斯的贡献改变了数学的根基，展示了函数可以比人们想象的更为复杂。

评论摘要
主要讨论点：数学中的反例，特别是关于连续性和可微性反例的讨论 不同观点： • seanhunter认为Dirichlet函数是微积分反例中的典范，因其在所有实数上定义但无处连续，且通过与x相乘可以构造出仅在0点连续的函数。这个例子展示了反常但有趣的数学性质。 • rtpg提到一本关于数学反例的法语书，指出很多连续性反例只是通过大量尝试得到的，虽然有趣但有些像作弊，并推荐这本书给说法语的人。 • thomassmith65分享了Weierstrass函数的Python实现，并好奇其音频表现，随后有人提供了音频示例链接。 • xelxebar提到了Devil's Staircase函数，这个函数在几乎所有地方都是常数但仍然能从0爬到1，展示了另一种反常的连续性特性。 • awanderingmind指出，许多数学家曾认为Weierstrass函数没有用，是数学中的异常现象，但实际上许多数学内容看似都是细枝末节。 • andrewhalle强调了连续但无处可微函数的普遍性，引用Baire Category Theorem说明大多数连续函数都是无处可微的，这与传统微积分中的函数形成对比。 • croes质疑讨论中是否提到了实际的函数，暗示对实际函数的讨论不足。 • graycat推荐了一本经典的反例书籍《Counterexamples in Analysis》，显示了其在数学分析中的重要性。 • cyberax分享了Schwarz lantern作为其最喜欢的数学反例，指出在三维中通过多面体逼近表面积的直觉是错误的。 • t234oi324324对反例的实用性提出质疑，认为基于19世纪分析的实数和Lebesgue积分与计算性不符，大多数实数在计算意义上是不可计算的，并暗示这可能导致数学失去关注。 补充讨论： • 讨论中涉及了多个具体的数学反例，如Dirichlet函数、Weierstrass函数、Devil's Staircase和Schwarz lantern，展示了这些反例在不同数学背景下的独特性质。 • 对于这些反例的实用性存在争议，一部分人认为这些反例虽然有趣但有些像作弊，另一部分人则认为它们展示了数学中重要的反常现象。 • t234oi324324的观点挑战了传统数学分析中反例的实用性，提出了计算性视角下的质疑，显示了对数学基础的不同看法。
