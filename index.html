How Unix spell ran in 64kb RAM

文章摘要

本文讲述了Unix系统中的拼写检查工具（Unix Spell）如何在1970年代资源极其有限的情况下，将250KB的词典压缩到64KB内存中并保持快速检索。面对PDP-11计算机的内存限制，Douglas McIlroy通过深入分析数据特性，开发了一种压缩算法，几乎达到了理论上的压缩极限。他首先使用了一种基于语言学的词干提取算法，将词典缩小到25,000词，并提高了准确性。为了加速检索，他起初使用了Bloom过滤器，后来由于词典扩大，他转向创新的哈希压缩技术，利用Golomb编码实现了每词13.60位的存储，接近理论最小值13.57位。通过分割压缩数据，他在略微增加内存使用的情况下大幅提升了检索速度。这一工程案例展示了在资源受限环境下如何通过数学洞察和精巧设计实现高效解决方案。

Show HN: Interactive systemd – a better way to work with systemd units

作者: kai-tub | 评分: 392 | 评论数: 28 | 发布时间: 2025-01-18 16:22
文章摘要

**主要内容总结：** `isd` 是一个用于简化 `systemd` 单元管理的工具，提供文本用户界面（TUI），支持模糊搜索、自动刷新预览、智能处理 `sudo` 权限，以及可自定义界面，适合新手和高级用户。它解决手动管理 `systemd` 单元的繁琐步骤，如启动、检查状态、查看日志和编辑单元等，通过自动刷新和统一界面展示相关信息和命令。 **安装方法：** 1. **AppImage**：下载并运行，适合任何Linux发行版。 2. **nix**：通过nix包管理器安装，可重复构建环境。 3. **uv**：通过Python包管理器安装，需Python 3.12及以上版本。 **功能特点：** - 模糊搜索 `systemd` 单元。 - 查看单元状态，自动刷新。 - 发送 `systemctl` 命令，查看详细日志。 - 自定义主题和界面。 `isd` 支持用户和系统模式，可切换展示不同类型的单元，并提供命令面板显示关键快捷键。

评论摘要
主要讨论点：围绕systemd及其工具的优缺点、使用体验和改进建议 不同观点： • **系统管理员对systemd命令的困扰**：[owyn] 认为systemd命令复杂且不直观，特别是启动和查看日志的命令（如`systemctl start`、`journalctl`）。目录结构（如`/lib/systemd/system`）也让传统Linux系统管理员感到不适，建议提升CLI的用户体验，使其更人性化。 • **ExecStop的混淆**：[ww520] 指出systemd的`ExecStop`命令容易引起误解，它是在服务启动命令完成后执行的，而不是在服务关闭时执行。要实现关闭时执行命令的功能，需要设置`RemainAfterExit=yes`。 • **调试服务的繁琐**：[wasted_intel] 表示使用原始CLI命令调试服务非常乏味，拥抱如lazygit等工具后效率提升显著，但对systemd的接口仍感到不满意。 • **工具安装问题**：[mahoro] 提到`uv`工具在Python 3.10环境下无法工作，需要使用`-p 3.13`选项。 • **远程主机连接需求**：[johnchristopher]、[yonatan8070] 关心工具是否支持连接远程主机以管理远程服务，当前systemd支持此功能。 • **多后端支持需求**：[throeurir] 希望工具能够支持本地安装并通过SSH和Docker执行命令。 • **安全问题**：[gchamonlive] 关注工具的安全性，特别是如何防止外部攻击者劫持控制。 补充讨论： • **工具兼容性问题**：[jchw] 讨论了工具在NixOS上的兼容性问题，指出在仅安装Nix的系统上可能也无法工作，并提供了替代的Nix命令建议。 • **工具的类比**：[jonwest] 将此工具与Kubernetes的k9s类比，认为其在便利性上相似。 争议焦点： • **systemd的易用性和接口设计**：部分用户认为systemd的CLI接口不直观、难以记忆，目录结构和命令设计不合理，尤其是在处理服务启动和停止时。 • **工具的安装和兼容性**：不同用户在安装工具时遇到了环境兼容性问题，特别是在NixOS和其他带有Nix的系统上。 总体而言，讨论围绕systemd的复杂性、命令记忆的困难、调试服务的繁琐以及工具在不同环境下的兼容性和功能扩展需求。

文章摘要

Intel's Skymont is the latest E-Core architecture, succeeding Crestmont in Meteor Lake chips, designed to boost multithreaded performance and handle background tasks more efficiently. In Lunar Lake, Skymont integrates two levels of E-Cores into one quad-core cluster on a low-power island, using the same TSMC N3B process node as P-Cores and improving cache hierarchy.
